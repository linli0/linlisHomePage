# 国内金价（黄金9999 · 元/克）设计

**日期:** 2026-07-19  
**状态:** 已确认并实现  
**范围:** FastAPI 金价服务 + 首页/金价页文案与刷新行为

---

## 目标

站内展示**国内人民币金价**，品种为上海黄金交易所相关公开行情中的 **黄金9999（AU9999）**，单位 **元/克**。  
更新频率：**每天自动刷一次** + **手动点击刷新**。不追求秒级实时。

## 非目标

- 不接入 MetalpriceAPI / 付费 Key
- 不做 WebSocket 推送
- 不做 60 秒轮询
- 本期不重构数据库列名（继续复用 `gold_prices.price_usd` 存「基准价」）

---

## 数据源

| 项 | 选择 |
|----|------|
| 源 | 新浪财经公开行情 `https://hq.sinajs.cn/list=gds_AU9999` |
| 请求头 | `Referer: https://finance.sina.com.cn`，常规 UA |
| 解析 | GBK 文本：`最新价,涨跌额,…,昨收,…,日期,名称`（以实测字段为准） |
| 单位 | 元/克 |
| 失败策略 | 保留库内上一笔有效价；打日志；手动刷新返回明确错误信息 |

备选（不实现，仅 fallback 文档）：东财 `push2.eastmoney.com`（字段需 ÷100）。

## 后端

### 存储语义

- `GoldPrice.price_usd`：**语义改为基准价 = AU9999 人民币元/克**（列名暂不改，避免迁移）
- `currency` 写入 `"CNY"`
- `change_amount` / `change_percent` 来自源数据（有则写，无则算相对昨收）
- 每次成功拉取 **新增一行**（便于历史）；`current` 取最新一行

### 读取 API（行为调整）

- `GET /api/gold-price/current?currency=CNY`（默认）：直接返回基准价，`symbol=¥`，文案侧单位为元/克
- 其他币种：用 `ExchangeRate` 从 CNY 估算（次要；前端默认 CNY）
- `GET /api/gold-price/history`：优先用库内按日记录；不足时用已有合成逻辑补齐（可后续再收紧）

### 刷新 API（新增）

- `POST /api/gold-price/refresh`
- 鉴权：与现网一致——若金价公开可读则 refresh 也可 `permitAll`（个人主页、一天一次）；若项目已要求登录写操作则跟现有惯例
- 行为：立即拉新浪 → 落库 → 返回最新 `GoldPriceDTO`
- 可选防护：同一进程内最短间隔 30s，避免连点打爆源站

### 日更定时

- 应用启动时注册后台任务（APScheduler 或 `asyncio` 循环均可；优先已有依赖，无则轻量 `threading.Timer` / lifespan 任务）
- Cron 风格：每天 **北京时间 09:30** 拉一次（交易日观感；周末拉到的仍是最近价，可接受）
- 启动后若库为空或最新价超过 24h：先刷一次再服务

## 前端

### 文案

- 「国际金价」→ **「国内金价」**
- 副标题/说明：**黄金9999 · 元/克**
- 价格旁单位明确为 **元/克**（不只显示 `¥`）

### 刷新

- 去掉金价页 60 秒自动轮询
- 「刷新」按钮调用 `POST /api/gold-price/refresh`，成功后更新卡片与「最后更新」时间
- 首页只读 `current`，不自动刷；用户进金价页点刷新即可

### API 封装

- `goldPriceApi.refresh()` → `POST /gold-price/refresh`

## 测试

- Mock 新浪响应，断言解析出价格与涨跌
- `POST /refresh` 成功写入后 `GET /current` 为新价
- 源失败时 `current` 仍返回旧价；`refresh` 返回非 200 或业务错误码

## 风险

| 风险 | 缓解 |
|------|------|
| 新浪接口改版/封禁 | 解析失败打日志；文档注明东财备选；UI 仍显示旧价 |
| 列名 `price_usd` 误导 | 代码注释 + 本设计文档说明 |
| 非交易时段价格不动 | 属预期；依赖「最后更新」时间戳说明 |

## 验收

1. 首页/金价页标题为「国内金价」，单位元/克，数量级约数百元/克（非美元盎司）
2. 点击刷新可更新价格与时间（网络可用时）
3. 不配置任何 API Key 即可工作
4. 无 60 秒轮询请求
