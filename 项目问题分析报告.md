## 🔴 一、安全问题（11 项 - 优先级：高）

### 1. 硬编码 JWT 密钥
- **文件**: `server.js:10`
- **问题**: JWT_SECRET 有弱默认值 `'coffee-cookies-secret-key-2024'`
- **影响**: 攻击者可伪造 JWT Token，完全绕过认证
- **修复建议**: 强制要求环境变量，不提供默认值

### 2. 硬编码默认密码
- **文件**: `server.js:18`, `project.properties`
- **问题**: 默认密码 `'admin'` 明文硬编码
- **影响**: 密码公开可见，无安全性
- **修复建议**: 移除默认密码，强制首次登录时设置

### 3. 密码明文比较
- **文件**: `server.js:188`
- **问题**: `if (password === SITE_PASSWORD)` 直接明文比较
- **影响**: 虽然导入了 bcrypt，但实际未使用，密码无哈希保护
- **修复建议**: 使用 bcrypt 哈希存储和验证密码

### 4. CORS 全开放
- **文件**: `server.js:42`
- **问题**: `app.use(cors())` 允许所有来源
- **影响**: CSRF 攻击风险
- **修复建议**: 限制允许的来源列表

### 5. JWT 存储在 localStorage
- **文件**: `request.ts:14`, `auth.ts:17`
- **问题**: Token 存储在 localStorage
- **影响**: XSS 攻击可窃取 Token
- **修复建议**: 使用 httpOnly Cookie 存储 Token

### 6. 无速率限制
- **问题**: 登录接口无防暴力破解保护
- **影响**: 攻击者可暴力破解密码
- **修复建议**: 实现 IP 级速率限制（如 express-rate-limit）

### 7. 弱默认数据库密码
- **文件**: `docker-compose.yml:48,51`
- **问题**: MySQL 默认密码 `root123`, `homepage123`
- **影响**: 数据库易被入侵
- **修复建议**: 强制环境变量，不提供默认值

### 8. 无输入过滤/净化
- **文件**: `server.js` 所有 POST 端点
- **问题**: 用户输入直接使用，无 sanitization
- **影响**: XSS 攻击、注入攻击风险
- **示例**: `/api/tools/json/format` 直接解析用户提供的 JSON
- **修复建议**: 使用 DOMPurify 或类似库过滤 HTML/JS

### 9. 缺少 HTTP 安全头
- **文件**: `server.js`
- **问题**: 无 CSP, X-Frame-Options, X-Content-Type-Options, HSTS 等安全头
- **影响**: XSS、点击劫持、MIME 嗅探攻击
- **修复建议**: 使用 helmet 中间件添加安全头

### 10. 无服务端错误日志
- **文件**: `server.js`
- **问题**: 错误仅返回客户端，无服务端记录
- **影响**: 无法监控和调查安全事件
- **修复建议**: 使用 winston 或 pino 记录所有错误

### 11. 密码修改功能不持久
- **文件**: `server.js:505-514`
- **问题**: 验证当前密码正确，但不实际更新 `SITE_PASSWORD` 或写入 `project.properties`
- **影响**: 用户以为密码已修改，实际未生效
- **修复建议**: 实现密码持久化逻辑

## 🟡 三、配置与部署问题（5 项 - 优先级：中）

### 15. 生产构建启用 Source Map
- **文件**: `frontend/vite.config.ts:24`
- **问题**: `sourcemap: true` 在生产构建中启用
- **影响**: 源代码暴露在 production，增加被逆向工程风险
- **修复建议**: 设置为 `sourcemap: false` 或 `sourcemap: 'hidden'`

### 16. Node 版本要求不一致
- **问题**:
  - `start.bat`: 要求 Node 18+
  - `frontend/Dockerfile`: 使用 `node:20-alpine`
  - `package.json`: 无 `engines` 字段指定 Node 版本
- **影响**: 环境不一致导致兼容性问题
- **修复建议**: 统一 Node 版本要求（建议 Node 20），在 package.json 添加 `engines` 字段

### 17. start.bat 不重新构建
- **文件**: `start.bat:34-53`
- **问题**: 只在 `frontend/dist` 不存在时构建，代码变更不触发重建
- **影响**: 部署过时前端代码
- **修复建议**: 每次启动前都执行 `npm run build` 或添加文件监视

### 18. 缺少 .env.example
- **问题**: 根目录无 `.env.example` 文件
- **影响**: 开发者不知道需要配置哪些环境变量
- **修复建议**: 创建 `.env.example` 文件，列出所有必需的环境变量

### 19. H2 控制台在生产启用
- **文件**: `backend/application.yml:23-26`
- **问题**: H2 控制台设置 `web-allow-others: true`
- **影响**: 数据库可被未授权访问
- **修复建议**: 在生产环境禁用 H2 控制台

## 🟡 四、代码质量问题（5 项 - 优先级：中）

### 20. TypeScript `any` 类型使用
- **文件**: `auth.ts`, `request.ts`, `ProfileView.vue:259`, `GoldPriceView.vue:268`
- **问题**: 部分地方使用 `any` 类型，如错误处理器参数和 catch 块
- **影响**: 丧失类型安全，增加运行时错误风险
- **修复建议**: 为所有参数和错误对象定义具体类型

### 21. 错误处理不完整
- **文件**: `HomeView.vue:426`, `GoldPriceView.vue:248`
- **问题**: 多处 `catch` 块只打印 console.error，无用户提示
- **影响**: 用户不知道操作失败，用户体验差
- **修复建议**: 向用户显示友好的错误消息

### 22. 导入顺序不规范
- **问题**: 部分文件未按 AGENTS.md 规定的顺序导入
- **要求**: Vue 核心 → 第三方库 → 本地组件 → 类型
- **修复建议**: 使用 ESLint 规则强制导入顺序

### 23. 魔法数字/字符串
- **问题**: 硬编码值如 `60000` (金价刷新间隔), `'USD'` (默认货币)
- **影响**: 代码可读性差，维护困难
- **修复建议**: 提取为命名常量

### 24. 不一致的错误处理
- **文件**: `server.js`
- **问题**: 部分端点使用 try/catch 返回 JSON 错误，部分依赖 Express 默认错误处理
- **影响**: 错误响应格式不一致，前端处理复杂
- **修复建议**: 统一使用全局错误处理中间件

## 🟡 五、前端数据流问题（4 项 - 优先级：中）

### 25. 金价数据重复获取
- **文件**: `HomeView.vue`, `GoldPriceView.vue`
- **问题**: 两个页面各自独立获取金价数据，无共享状态
- **影响**: 多余 API 调用，数据可能不同步
- **修复建议**: 使用 Pinia Store 统一管理金价状态

### 26. 无数据缓存策略
- **问题**: 每次路由切换都重新获取数据
- **影响**: 用户体验差，API 负载高
- **修复建议**: 实现客户端缓存（如使用 SWR 或 React Query 的 Vue 等价物）

### 27. 组件间通信低效
- **问题**: 部分数据通过 props 层层传递（prop drilling）
- **影响**: 组件耦合度高，难以维护
- **修复建议**: 使用 Pinia Store 或 provide/inject 共享状态

### 28. AI 流式响应使用 fetch 而非 Axios
- **文件**: `ai.ts:33-110`
- **问题**: 流式响应直接用 fetch，与项目 Axios 封装不一致
- **影响**: 代码风格不统一，JWT 注入逻辑重复
- **修复建议**: 为 Axios 添加流式响应支持或统一使用 fetch

## 🟡 六、性能问题（2 项 - 优先级：中）

### 29. 金价历史数据无缓存
- **文件**: `server.js:134-149`
- **问题**: `getStats()` 每次请求都调用 `generateHistory(30, goldPrice[currency].price)` 重新计算
- **影响**: 不必要的 CPU 开销
- **修复建议**: 缓存计算结果，定期刷新

### 30. 无数值参数输入验证
- **文件**: `server.js`
- **问题**:
  - 金价历史端点接受 `days` 参数无验证
  - 二维码端点接受 `width`/`height` 参数无边界检查
- **影响**: DoS 攻击风险（如 `days=999999999` 导致内存耗尽）
- **修复建议**: 添加参数范围验证（如 days 最大 365）
